# 欧加云控配置解析

> 本文以一加 15（SM8850）的配置为例，解析云控各字段的含义与作用。

---

## 基础信息

```json
"package_name": "com.tencent.tmgp.cf"
```

> **应用包名** — 标识此云控配置所针对的游戏应用。

```json
"type_id": null,
"feature_flag": "72354321"
```

> **云控下发的应用 ID** — 用于区分不同云控策略的唯一标识。

---

## CPU 核心信息与频率表

> 此设备的 CPU 分为两个集群，各字段中涉及的频率索引数字均参照下表。**频率索引从 0 开始**，数字与频率一一对应。

### 集群概览

| 集群 | 包含核心 | 核心类型 | 频率档位数 | 最低频率 | 最高频率 |
|------|---------|---------|-----------|---------|---------|
| `policy0` | 0, 1, 2, 3, 4, 5 | 小核 + 大核 | 28 档 | 384 MHz | 3629 MHz |
| `policy6` | 6, 7 | 超大核 | 28 档（部分未启用） | 768 MHz | 4608 MHz |

> **当前调速器**: 两集群均为 `walt`
>
> **支持的调速器**: `conservative`, `hmbird`, `performance`, `powersave`, `schedutil`, `walt`

### policy0 频率表（核心 0-5）

| 索引 | 频率 | 索引 | 频率 | 索引 | 频率 | 索引 | 频率 |
|:---:|------|:---:|------|:---:|------|:---:|------|
| 0 | 384 MHz | 7 | 1229 MHz | 14 | 1997 MHz | 21 | 2899 MHz |
| 1 | 538 MHz | 8 | 1325 MHz | 15 | 2112 MHz | 22 | 3034 MHz |
| 2 | 691 MHz | 9 | 1440 MHz | 16 | 2227 MHz | 23 | 3187 MHz |
| 3 | 787 MHz | 10 | 1555 MHz | 17 | 2362 MHz | 24 | 3302 MHz |
| 4 | 883 MHz | 11 | 1670 MHz | 18 | 2496 MHz | 25 | 3398 MHz |
| 5 | 998 MHz | 12 | 1786 MHz | 19 | 2611 MHz | 26 | 3514 MHz |
| 6 | 1114 MHz | 13 | 1901 MHz | 20 | 2746 MHz | 27 | **3629 MHz** |

### policy6 频率表（核心 6-7）

| 索引 | 频率 | 索引 | 频率 | 索引 | 频率 | 索引 | 频率 |
|:---:|------|:---:|------|:---:|------|:---:|------|
| 0 | 768 MHz | 7 | 1632 MHz | 14 | 2669 MHz | 21 | 3974 MHz |
| 1 | 883 MHz | 8 | 1747 MHz | 15 | 2880 MHz | 22 | 4070 MHz |
| 2 | 1018 MHz | 9 | 1862 MHz | 16 | 3130 MHz | 23 | 4186 MHz |
| 3 | 1133 MHz | 10 | 1978 MHz | 17 | 3398 MHz | 24 | 4262 MHz |
| 4 | 1267 MHz | 11 | 2112 MHz | 18 | 3648 MHz | 25 | 4339 MHz |
| 5 | 1382 MHz | 12 | 2227 MHz | 19 | 3763 MHz | 26 | 4397 MHz |
| 6 | 1498 MHz | 13 | 2438 MHz | 20 | 3859 MHz |    | **4608 MHz**<sup>max</sup> |

> 硬件最高频率为 **4608 MHz**，但频率表仅列出到索引 26（4397 MHz），其余档位由硬件动态管理。

### 快速查阅示例

例如配置中出现 `"0,27,6,28"`（`cht_boost_max`），表示：

| 字段 | 集群 | 频率索引 | 对应频率 |
|------|------|:--------:|---------|
| 前两个数 `0,27` | policy0 | 索引 27 | **3629 MHz**（policy0 最高档） |
| 后两个数 `6,28` | policy6 | 索引 28 | **4608 MHz**（policy6 最高档） |

## CPU 配置 — `cpu_config`

不同场景下的 CPU 升频配置。

```json
"cpu_config": {
  "start": {
    "boost": "10,27,15,28",
    "time": 5000
  }
}
```

| 场景键名 | 说明 |
|---------|------|
| `start` | 打开游戏 |
| `logging` | 游戏登录 |
| `chooserole` | MOBA 游戏选择英雄 |
| `loading` | 加载中 |
| `sceneswitch` | 场景切换（如原神等游戏） |
| `landing` | 落地（如吃鸡类游戏） |
| `parachute` | 跳伞降落 |

### 字段说明

| 字段 | 说明 |
|------|------|
| `boost` | **升频上下限** — 四个数字为一组，格式：`大核下限,大核上限,超大核下限,超大核上限`。数字与 CPU 频率表一一对应，从 0 开始。 |
| `time` | **升频持续时间**（单位：ms） |

## 刷新率设置 — `refresh_rate`

```json
"refresh_rate": {
  "touch_rate": { ... }
}
```

### 采样率设置 — `touch_rate`

各性能模式下，不同帧率对应的**触控采样率**。数字代表不同的采样率等级。

| 帧率 | 省电模式 | 均衡模式 | 性能模式 | X 模式 |
|------|---------|---------|---------|--------|
| 60   | 2       | 2       | 17      | 17     |
| 90   | 2       | 2       | 17      | 17     |
| 120  | 2       | 2       | 17      | 17     |
| 144  | 12      | 12      | 17      | 17     |
| 165  | 13      | 13      | 17      | 17     |

> 模式映射：`energySavingTouchRateLevel` → 省电模式，`touchRateLevel` → 均衡模式，`perfLevel` → 性能模式，`xModeLevel` → X 模式。

## GPA 调度配置 — `gpa_config`

游戏内不同帧率对应的调度相关设置，以 120 帧为例：

```json
"gpa_config": {
  "120": {
    "cl": 0,
    "ch": 27,
    "gm": 28,
    "gf": 0,
    "mtl": "1;30:88;100:90,1;33:70;50:72;100:75",
    "dpcpus": "0-5",
    "sfcpus": "0-5",
    "smt": 2,
    "bpf": 1,
    "sync": 1,
    "chtb": { "enable": true },
    "es4g": { "state": true, "fps": 120, "resv": "0xc0", "phr": "85,100", "coe": "4096,0" },
    "geas": { ... },
    "mema": { ... },
    "cscene": { "cfg": { "bpf": 1 } }
  }
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `cl` | **大核最小频率** |
| `ch` | **大核最大频率** |
| `gm` | **超大核最大频率** |
| `gf` | **超大核最小频率** |
| `mtl` | 一般不动，保持默认即可 |
| `dpcpus` | **用户后台进程**分配的核心范围 |
| `sfcpus` | **系统后台进程**分配的核心范围 |
| `chtb` | **升频开关** — 下方升频率相关内容依赖此开关开启 |
| `es4g` | **风驰调速器** — scx、hmbird 启用状态 |
| `geas` | 详细调度参数，包含 `param0`、`param1` 和 `algorithm` |
| `mema` | 内存/频率调度策略（见下方详述） |
| `cscene` | 场景配置 |

数字分别与核心频率表频率一一对应。

### es4g 风驰调速器

| 字段 | 说明 |
|------|------|
| `state` | 启用状态 |
| `fps` | 目标帧率 |
| `resv` | 保留参数 |
| `phr` | 百分比阈值 |
| `coe` | 系数 |

### geas 调度参数

```json
"geas": {
  "param0": {
    "flag": 4,
    "frdr": "6,0,1,100,100,1,1,100,100,1,1,1,1,10,0,0,125,40,2136,5285,6074,6671,9343,11468,13230,14398,16500,18597"
  },
  "param1": {
    "flag": 4,
    "frdr": "6,1,1,100,100,1,1,100,100,1,1,1,1,10,0,0,125,40,2136,5285,6074,6671,9343,11468,13230,14398,16500,18597"
  },
  "algorithm": "-1,-1,-1,1,-1,-1,-1"
}
```

| 字段 | 说明 |
|------|------|
| `flag` | 参数标志位 |
| `frdr` | 频率调度数据序列 |
| `algorithm` | 算法选择，`1` 表示启用对应算法 |

### mema 内存调度策略

```json
"mema": {
  "beta": "35",
  "mode": "0",
  "fl": 1,
  "custom": {
    "4-5":  { "beta": 35, "mode": 0, "tl": "0,80,300,82,450,85,600,88" },
    "0-3":  { "beta": 50, "mode": 0, "tl": "0,85,300,88,450,90,600,92" },
    "6-7":  { "beta": 30, "mode": 0, "tl": "0,74,500,74,700,76,900,78" }
  }
}
```

| 字段 | 说明 |
|------|------|
| `beta` | **激进程度** — 数字越大越激进，不要无脑放大 |
| `mode` | **模式** — `0` 默认，`2` 可能会更激进 |
| `fl` | 标志位 |
| `tl` | 阈值+频率序列，格式：`负载,频率,负载,频率, ...` |

核心分组说明：

| 核心组 | CPU 核心 | 作用 |
|--------|---------|------|
| `0-3` | 大核 | 常用负载较高 |
| `4-5` | 大核 | 中等激进 |
| `6-7` | 超大核 | 保守策略 |

## 线程设置 — `game_zone`

```json
"game_zone": {
  "recognize_interval": "1000",
  "wake_up_times": "10",
  "key_thread_ux": "2",
  "key_worker_ux": "2",
  "white_list_ux": "2",
  "white_list": [
    "tencent.tmgp.cf",
    "AudioTrack",
    "FMOD Mixer Thre"
  ],
  "pipeline": {
    "4": "UnityChoreograp",
    "6": "UnityGfxDeviceW",
    "7": "UnityMain"
  },
  "bind_core": "1",
  "bind_list": {
    "UnityMain": "g_11",
    "UnityGfxDeviceW": "g_10",
    "Thread-:2": "p_4"
  },
  "preempt": [
    "__render__",
    "UnityMain",
    "GPU completion",
    "UnityChoreograp"
  ]
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `recognize_interval` | 场景识别间隔（ms） |
| `wake_up_times` | 唤醒次数 |
| `key_thread_ux` | 关键线程 UX 优先级 |
| `key_worker_ux` | 关键工作线程 UX 优先级 |
| `white_list_ux` | 白名单线程 UX 优先级 |
| `white_list` | **白名单线程** — 自动绑定 0-7 核心 |
| `pipeline` | **线程亲和性** — 将特定线程绑定到指定核心 |
| `bind_core` | 是否启用核心绑定 |
| `bind_list` | **线程→核心绑定映射** |
| `preempt` | **抢占列表** — 优先调度的线程 |

### 核心绑定语法

| 前缀 | 含义 |
|------|------|
| `g_` | 指定物理核心编号 |
| `p_` | 指定核心组策略 |

## 游戏全局设置 — `game_config`

优先级比上述部分参数更高。

```json
"game_config": {
  "ctb": 1,
  "htb": 1,
  "hbs": 1,
  "ctep": 105,
  "ctn": "UnityMain UnityGfxDeviceW",
  "cht_boost_max": "0,27,6,28",
  "cht_boost_min": "0,11,6,14",
  "scaling_min": "0,21,6,16",
  "scaling_max": "0,21,6,16",
  "reset_scaling_min": "0,3,6,1",
  "reset_scaling_max": "0,27,6,28",
  "fsd": "3",
  "ssd": "2",
  "tfps": "185",
  "nsu": "2",
  "nsd": "3",
  "nlru": "1",
  "nlrd": "1",
  "rgr": 1,
  "perfd": { "status": true, "sde": 0 }
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `ctb` | 通用调速器 boost 开关 |
| `htb` | 硬件加速 boost 开关 |
| `hbs` | 硬件加速调度开关 |
| `ctep` | 效率阈值 |
| `ctn` | 关键线程名称列表 |
| `cht_boost_max` | 升频最大值（格式：`policy0,值,policy6,值`） |
| `cht_boost_min` | 升频最小值 |
| `scaling_min` | **全局最小频率** — `0/6` 代表 policy0/policy6，后面的数字对应频率表 |
| `scaling_max` | **全局最大频率** — `0/6` 代表 policy0/policy6，后面的数字对应频率表 |
| `reset_scaling_min` | **返回桌面时**设置的最小频率 |
| `reset_scaling_max` | **返回桌面时**设置的最大频率 |
| `fsd` | FPS 稳定下降步长 |
| `ssd` | 场景切换下降步长 |
| `tfps` | 目标帧率上限（如 `185`） |
| `nsu` | 核心上线步长 |
| `nsd` | 核心下线步长 |
| `nlru` | 负载上报上调步长 |
| `nlrd` | 负载上报下调步长 |
| `rgr` | 调速器再评估开关 |
| `perfd` | 性能描述符设置 |

### scaling 格式说明

```
<policy>,<频率索引>,<policy>,<频率索引>, ...
```

- `0` = policy0（小/大核集群）
- `6` = policy6（超大核集群）
- 后面的数字对应 CPU 频率表中的索引位置（从 0 开始）

## 温控锁帧 — `thermal_frame`

不同帧率下，按温度和性能模式锁定更低帧率的策略。

```json
"thermal_frame": {
  "165": {
    "balance_nl": "46,120,46,90,46,60",
    "highperf_nl": "46,120,48,90,49,60",
    "ternary": false
  },
  "120": {
    "balance_nl": "48,90,48,60",
    "highperf_nl": "48,90,49,60"
  },
  "90": {
    "balance_nl": "46,60",
    "highperf_nl": "49,60"
  },
  "185": {
    "balance_nl": "48,165,48,90,46,60",
    "highperf_nl": "48,165,48,90,49,60",
    "ternary": false
  }
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `balance_nl` | **均衡模式** — 温度阈值与锁定帧率配对：`温度,锁定帧率,温度,锁定帧率, ...` |
| `highperf_nl` | **性能模式** — 格式同上 |
| `ternary` | 是否启用三级降帧（true/false） |

> 示例：`"46,120,46,90,46,60"` 表示 **46°C 锁 120 帧、46°C 锁 90 帧、46°C 锁 60 帧**，即温度达到 46°C 时依次降级。

## FPS 稳定器 — `fps_stabilizer`

涉及升频率的稳定策略。

```json
"fps_stabilizer": {
  "boostStep": "1.1,1.5,1.5",
  "freqStep": "24,7,24,13,25,10,25,16,28,13,28,15",
  "temp": 490,
  "boostTime": 500
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `boostStep` | **Boost 等级** — 三组值，与 `freqStep` 的三组一一对应 |
| `freqStep` | **频率步进** — 四个数为一组，对应：`大核上限,大核下限,超大核上限,超大核下限`；共三组 |
| `temp` | 温控阈值 |
| `boostTime` | Boost 持续时间（单位：ms） |

### freqStep 分组对照

| 组 | 大核上限 | 大核下限 | 超大核上限 | 超大核下限 | 对应 Boost 等级 |
|---|---------|---------|-----------|-----------|----------------|
| 1 | 24 | 7 | 24 | 13 | 1.1 |
| 2 | 25 | 10 | 25 | 16 | 1.5 |
| 3 | 28 | 13 | 28 | 15 | 1.5 |
